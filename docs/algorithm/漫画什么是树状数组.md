树状数组是一种支持 **单点修改** 和 **区间查询** 的，代码量小的数据结构。

事实上，树状数组能解决的问题是线段树能解决的问题的子集：树状数组能做的，线段树一定能做；线段树能做的，树状数组不一定可以．然而，树状数组的代码要远比线段树短，时间效率常数也更小，因此仍有学习价值．

我们学习数据结构的目的在于将我们的算法变得更快。由 Peter M. Fenwick 提出的树状数组 **BIT** 结构（Fenwick 树）就是一个优秀的数据结构，**BIT** 全称 **Binary Indexed Trees** 结构，而不是所说的比特奥。Peter M. Fenwick 首次使用此结构进行**<font style="color:#D22D8D;">数据压缩</font>**。在算法竞赛中，通常用于 **<font style="color:#D22D8D;">存储频率 和 处理累积频率表</font>**。

首先考虑一个简单的问题。

给定一个数组 `arr[0 ... n-1]` ，如何实现下面<font style="color:#ED740C;">两个操作</font>：

1. 计算前 `i` 个元素的累加和；
2. 将数组中下标为 `i` 的元素的值更新为 `x`，`arr[i] = x` ，其中 `0 <= i <= n-1`

一个简单的方法就是遍历 0 到 `i - 1` 的元素并计算出累加和即可 ；然后更新操作 `arr[i] = x` 就可以直接进行，也就说可以对数组 `arr[]` 直接进行修改.

```cpp
// 计算前 i 个元素的累加和
public int getSum(int arr[], int i){
    int sum = 0;
    for(int j = 0; j < i; j++){
        sum += arr[j];
    }
    return sum;
}
```

这种方式第一个操作，也就是计算累加和的时间复杂度为$ O(n) $ ，更新操作的时间复杂度为$ O(1) $ ;

另外一种方式就是创建一个大小为 n 的新数组，并且在**新数组的第 **`**i**`** 个位置保存前 **`**i**`** 个元素的累加和**。此时查找给定范围内的累加和就可以在$ O(1) $ 的时间内完成，但是更新操作将花费$ O(n) $ 的时间，<font style="color:#D22D8D;">这对于大量的查询操作，而更新操作比较少的问题很实用</font>。

```cpp
// 更新数组 arr[i] = x 之后
// 需要对存储累加和的数组 new_arr 进行的修改。
void updateSum(int arr[], int i, int x){
    arr[i] = x;
    for(int j = i; j < new_arr.length; j++){
        new_arr[j] = new_arr[j-1] + arr[j];
    }
}
```

也就说，要实现上面提到的两个操作，要么查找为$ O(n) $ ，更新操作为$ O(1) $ ；要么使用额外的空间，将查找操作降为$ O(1) $ ，但是更新操作变为了$ O(n) $ .

## 树状数组
### 那么是否可以将查找和更新操作同时降低到$ O(logn) $ 呢？
一个就是以后会讲到的**<font style="color:#ED740C;">线段树（Segment Tree）</font>**，另外一个就是树状数组 （Binary Indexed Tree），两者均可以将上面所提到的查找和更新操作的时间复杂度降到 。但是<font style="color:#D22D8D;">与线段树相比，树状数组的效率更高</font>，并且易于实现。

树状数组表示为 `BITree[]`；树状数组的每个节点存储输入数组中某些元素的和；树状数组的大小等于输入数组的大小，记作 n 。为了便于实现，`BITree[]` 使用 n+1 的大小。

### 如何根据 `BITree[]` 树状数组，获取数组 `arr[]` 中前 `i` 个元素的累加和？
这里更关键奥！！！

我们都知道，<font style="color:#D22D8D;">任何一个正整数都可以被表示为 2 的次幂和</font>，比如 `11` 可以表示为 8 + 2 + 1.** **`**BITree**`**的每个节点都存储 n 个元素的总和，其中 n 是 2 的次幂**。比如前 11 个元素的累加和可以通过对原数组 `arr[]` 中最后 1 个元素（第11个元素）、向前两个元素（第 9 和 10 号元素）和 前 8 个元素 （从 1 到 8 的）的元素之和求得。

<!-- 这是一张图片，ocr 内容为： -->
![](https://images.spumn.eu.cc/fenwick-20.png)

对照上图，来理解文字描述就更清晰了，<font style="color:#ED740C;">我们求前 11 个元素的累加和，可以将其分解为 2 的次幂之和，即 8 + 2 + 1，也就是前 8 个元素的累加和（1 到 8），紧挨着的 2 个元素（9 和 10），和最后 1个元素 （11）</font>三者的和。

如果从树状数组的角度来看，`**BITree[8] = 21**` 表示前 8 个元素的累加和，`**BITree[10] = 13**` 表示 6 和 7 的和（这里解释一下， $ 10 \&(-10) = 2 $ 表示的就是两个数的和）， `**BITree[11] = 8**` 表示一个 8 （$ 11 \&(-11) = 2 $，表示 1 个数的和） 。所以**前 11 个**元素的累加和等于 `**BITree[8] + BITree[10] + BITree[11] = 21 + 13 + 8 = 42**` 。

如果再从更直观的树上看，**<font style="color:#601BDE;">计算</font>****<font style="color:#DF2A3F;">前 11 个</font>****<font style="color:#601BDE;">元素的累加和，从叶子结点 11 开始，找到 11 的父结点 10，然后找到 10 的父结点 8 ，8 的父结点为 0 </font>**，然后将路径上的值都加起来，就是前 11 个元素的累加和。

不难写出下面计算累加和的代码：

```cpp
int getSum(int index) 
{ 
    int sum = 0; // 累加和

    // BITree[] 的下标比 arr[] 大 1
    index = index + 1; 

    // 遍历 BITree[index] 的祖先结点
    while(index>0) 
    { 
        // 将当前 BITree 的值加到 sum
        sum += BITree[index]; 

        // 将 index 指向 index 的父结点
        index = index - index & (-index); 
    } 
    return sum; 
}
```

代码很清晰，就是从给定的 index 遍历 index 的所有的祖先结点，并将遍历到的 `BITree[index]` 的值加起来即可。

代码也相当简单：

```cpp
public static void updateBIT(int n, int index, int val) 
{ 
    // BITree[] 的下标比 arr[] 大 1
    index = index + 1; 

    // 遍历所有的祖先，并加上 'val'
    while(index <= n) 
    { 
        // BIT Tree 的当前结点加上 'val'
        BITree[index] += val; 

        // 更新 index
        index += index & (-index); 
    } 
}
```

### 能否将树状数组扩展到以$ O(logn) $的时间复杂度计算<font style="color:#D22D8D;">区间和</font>呢？
答案是肯定的，`rangSum(l,r) = getSum(r) - getSum(l - 1)` .

## 复杂度分析
任何一个正整数 n 的二进制表示中置位数的个数为$ O(nlogn) $ 量级，<font style="color:#DF2A3F;">置位数就是一个整数二进制表示中 1 的数目</font>。因此，`getSum()` 和 `updateBIT()` 两个操作至多遍历$ O(logn) $ 个结点。

初始构造树状数组 `BITree[]` 的时间复杂度为$ O(nlogn) $，构造 `BITree[]` 树状数组会调用 `updateBIT()` 函数 n 次。

## 完整的实现代码
```java
import java.util.*; 
import java.lang.*; 
import java.io.*; 

class BinaryIndexedTree 
{ 
    final static int MAX = 100;  
    static int BITree[] = new int[MAX]; 

    int getSum(int index) 
    { 
        int sum = 0;
        index = index + 1; 

        while(index>0) 
        { 
            sum += BITree[index]; 
            index -= index & (-index); 
        } 
        return sum; 
    } 

    public static void updateBIT(int n, int index, 
                                 int val) 
    { 
        index = index + 1; 

        while(index <= n) 
        { 
            BITree[index] += val; 
            index += index & (-index); 
        } 

    } 

    void printBITree(int arr[], int n) {
        for(int i = 0; i < n; i++) {
            System.out.print(arr[i] + " ");
        }
        System.out.println();
    }
    
    void constructBITree(int arr[], int n) 
    { 
        for(int i=1; i<=n; i++) 
        	BITree[i] = 0; 
        for(int i = 0; i < n; i++) {
            updateBIT(n, i, arr[i]); 
            printBITree(BITree,n+1);
        }

    } 

    public static void main(String args[]) 
    { 
        int arr[] = {2, 1, 1, 3, 2, 3, 4, 5, 6, 7, 8, 9}; 
        int n = arr.length; 
        BinaryIndexedTree tree = new BinaryIndexedTree(); 

        // 从给定的数组 arr[], 构造 BITree[]
        tree.constructBITree(arr, n); 

        System.out.println("arr[0..5] = " + tree.getSum(5)); 

        // 测试更新操作
        arr[3] += 6; 

        // arr[3] 的改变，更新 BITree[]
        updateBIT(n, 3, 6);  

        System.out.println("arr[0..5] = " + tree.getSum(5)); 
    }
```
