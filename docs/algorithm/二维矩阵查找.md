# 题目描述


LeetCode [240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/)



编写一个高效的算法来搜索 `m x n` 矩阵 `matrix` 中的一个目标值 `target` 。该矩阵具有以下特性：



+ 每行的元素从左到右升序排列。
+ 每列的元素从上到下升序排列。



**示例 1：**



![](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2020/11/25/searchgrid2.jpg)



> 输入：matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 5  
输出：true
>



**示例 2：**



![](https://assets.leetcode-cn.com/aliyun-lc-upload/uploads/2020/11/25/searchgrid.jpg)



> 输入：matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 20  
输出：false
>



**提示：**



+ `m == matrix.length`
+ `n == matrix[i].length`
+ `1 <= n, m <= 300`
+ `-10^9 <= matrix[i][j] <= 10^9`
+ 每行的所有元素从左到右升序排列
+ 每列的所有元素从上到下升序排列
+ `-10^9 <= target <= 10^9`



# 方法一：一维二分查找


## 思路与算法


由于矩阵 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D) 中每一行的元素都是升序排列的，因此我们可以对每一行都使用一次二分查找，判断 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D) 是否在该行中，从而判断 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D)是否出现。



**代码**



```c
class Solution {
public:
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        for (const auto& row: matrix) {
            auto it = lower_bound(row.begin(), row.end(), target);
            if (it != row.end() && *it == target) {
                return true;
            }
        }
        return false;
    }
};
```



## **复杂度分析**


+  时间复杂度：![image](https://g.yuque.com/gr/latex?O(m%20%5Clog%20n))。对一行使用二分查找的时间复杂度为 ![image](https://g.yuque.com/gr/latex?O(%5Clog%20n))，最多需要进行 ![image](https://g.yuque.com/gr/latex?m) 次二分查找。 
+  空间复杂度：![image](https://g.yuque.com/gr/latex?O(1))。 



# 方法二：Z 字形线性查找


假设现有矩阵 matrix 如下：



```c
[
  [1,   4,  7, 11, 15],
  [2,   5,  8, 12, 19],
  [3,   6,  9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
]
```



假设我们需要查找`matrix[1][2] = 8`, 由于每一行的元素都是升序排列的，每一列的元素都是升序排列的，那么：



右下角的数一定都比8大：



```bash
[
  [8, 12, 19],
  [9, 16, 22],
  [14, 17, 24],
  [23, 26, 30]
]
```



左上角的数一定都比8小：



```c
[
  [1,   4,  7],
  [2,   5,  8]
]
```



故而这两块都可以直接排除。



那么剩下的左下角的数和右上角的数与8的大小关系不知道如何，故而只需要验证这两块即可：



```c
//右上角
[
  [7, 11, 15],
  [8, 12, 19]
]

//左下角
[
  [2,   5,  8],
  [3,   6,  9],
  [10, 13, 14],
  [18, 21, 23]
]
```

我们可以看到，**矩阵中大于 mid 的数就和不大于 mid 的数分别形成了两个板块，沿着一条锯齿线将这个矩形分开**。其中左上角板块的大小即为矩阵中不大于 mid 的数的数量。

![](https://cdn.nlark.com/yuque/0/2022/png/22382307/1667721470659-840a6dbc-813d-4354-8975-0faf322ab2ae.png)

考虑子问题，我们每一次都选择从**右上角开始遍历**，这样一来就可以排除右上角这一块的干扰。从而将问题转化为以 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D) 的左下角为左下角、以 ![image](https://g.yuque.com/gr/latex?(x%2C%20y))为右上角的矩阵新的子问题：



```c
[
  [2,   5,  8],
  [3,   6,  9],
  [10, 13, 14],
  [18, 21, 23]
]
```



以右上角为起点开始逐渐缩小矩阵，这样每次只会考虑左下角这一块矩阵即可：



+ 如果`x < target`，那么`x`左边的所有数字都可以排除掉。
+ 如果`x > target`，那么`x`下边的所有数字都可以排除掉。



## **思路与算法**


我们可以从矩阵 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D) 的右上角 ![image](https://g.yuque.com/gr/latex?(0%2C%20n-1))进行搜索。在每一步的搜索过程中，如果我们位于位置 ![image](https://g.yuque.com/gr/latex?(x%2C%20y))，那么我们希望在以 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D) 的左下角为左下角、以 ![image](https://g.yuque.com/gr/latex?(x%2C%20y))为右上角的矩阵中进行搜索，即行的范围为$ [x, m - 1]![image](https://g.yuque.com/gr/latex?%EF%BC%8C%E5%88%97%E7%9A%84%E8%8C%83%E5%9B%B4%E4%B8%BA) [0, y]$：



+ 如果 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D%5Bx%2C%20y%5D%20%3D%20%5Ctextit%7Btarget%7D)，说明搜索完成；
+ 如果 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D%5Bx%2C%20y%5D%20%3E%20%5Ctextit%7Btarget%7D)，由于每一列的元素都是升序排列的，那么在当前的搜索矩阵中，所有位于第 ![image](https://g.yuque.com/gr/latex?y) 列的元素都是严格大于 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D) 的，因此我们可以将它们全部忽略，即将 ![image](https://g.yuque.com/gr/latex?y) 减少 ![image](https://g.yuque.com/gr/latex?1)；
+ 如果 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Bmatrix%7D%5Bx%2C%20y%5D%20%3C%20%5Ctextit%7Btarget%7D)，由于每一行的元素都是升序排列的，那么在当前的搜索矩阵中，所有位于第 ![image](https://g.yuque.com/gr/latex?x) 行的元素都是严格小于 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D)的，因此我们可以将它们全部忽略，即将 ![image](https://g.yuque.com/gr/latex?x) 增加 ![image](https://g.yuque.com/gr/latex?1)。



在搜索的过程中，如果我们超出了矩阵的边界，那么说明矩阵中不存在 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D)。



**代码**



```c
class Solution {
public:
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        int m = matrix.size(), n = matrix[0].size();
        int x = 0, y = n - 1;
        while (x < m && y >= 0) {
            if (matrix[x][y] == target) {
                return true;
            }
            if (matrix[x][y] > target) {
                --y;
            }
            else {
                ++x;
            }
        }
        return false;
    }
};
```



## **复杂度分析**


+  时间复杂度：![image](https://g.yuque.com/gr/latex?O(m%20%2B%20n))。在搜索的过程中，如果我们没有找到 ![image](https://g.yuque.com/gr/latex?%5Ctextit%7Btarget%7D)，那么我们要么将 ![image](https://g.yuque.com/gr/latex?y) 减少 ![image](https://g.yuque.com/gr/latex?1)，要么将 ![image](https://g.yuque.com/gr/latex?x) 增加 ![image](https://g.yuque.com/gr/latex?1)。由于 ![image](https://g.yuque.com/gr/latex?(x%2C%20y)) 的初始值分别为 ![image](https://g.yuque.com/gr/latex?(0%2C%20n-1))，因此 ![image](https://g.yuque.com/gr/latex?y) 最多能被减少 ![image](https://g.yuque.com/gr/latex?n) 次，![image](https://g.yuque.com/gr/latex?x) 最多能被增加 ![image](https://g.yuque.com/gr/latex?m) 次，总搜索次数为 ![image](https://g.yuque.com/gr/latex?m%20%2B%20n)。在这之后，![image](https://g.yuque.com/gr/latex?x) 和 ![image](https://g.yuque.com/gr/latex?y) 就会超出矩阵的边界。 
+  空间复杂度：![image](https://g.yuque.com/gr/latex?O(1))。
