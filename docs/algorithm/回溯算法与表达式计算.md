# 回溯算法与表达式计算

[282. Expression Add Operators](https://leetcode.cn/problems/expression-add-operators/)

Given a string num that contains only digits and an integer target, return all possibilities to insert the binary operators '+', '-', and/or '*' between the digits of num so that the resultant expression evaluates to the target value.

Note that operands in the returned expressions should not contain leading zeros.

Example 1:

Input: num = &quot;123&quot;, target = 6
Output: [&quot;`1*2*3`&quot;,&quot;`1+2+3`&quot;]
Explanation: Both &quot;1*2*3&quot; and &quot;1+2+3&quot; evaluate to 6.

Example 2:

Input: num = &quot;232&quot;, target = 8
Output: [&quot;`2*3+2`&quot;,&quot;`2+3*2`&quot;]
Explanation: Both &quot;`2*3+2`&quot; and &quot;`2+3*2`&quot; evaluate to 8.

Example 3:

Input: num = &quot;3456237490&quot;, target = 9191
Output: []
Explanation: There are no expressions that can be created from &quot;3456237490&quot; to evaluate to 9191.

# 回溯思路

设字符串 1813718137 的长度为 n，为构建表达式，我们可以往 1813718137 中间的 1813718137 个空隙添加 1813718137 号、1813718137 号或 1813718137号，或者不添加符号。

我们可以用「回溯法」来模拟这个过程。从左向右构建表达式，并实时计算表达式的结果。由于乘法运算优先级高于加法和减法运算，我们还需要保存最后一个连乘串（如 1813718137）的运算结果。

这里困难的是如何处理优先级运算1813718137。我们可以每次记录当前结果1813718137和最后计入(1813718137) 1813718137的连乘串1813718137。这样万一下面还是1813718137号，则**撤销**上一步1813718137计算结果，重新加上1813718137的结果作为1813718137，从而达到**直接按从左到右的顺序求出有乘法的表达式。**

定义递归函数 1813718137，其中：

-  1813718137 为当前构建出的表达式； 
-  1813718137 表示当前的枚举到了 1813718137 的第 $i $个数字； 
-  1813718137为当前表达式的计算结果； 
-  1813718137 为表达式最后一个连乘串的计算结果。 

该递归函数分为两种情况：

如果 1813718137，说明表达式已经构造完成，若此时有 1813718137，则找到了一个可行解，我们将 1813718137放入答案数组中，递归结束；

如果 1813718137，需要枚举当前表达式末尾要添加的符号（1813718137 号、1813718137 号或 1813718137 号），以及该符号之后需要截取多少位数字。设该符号之后的数字为 1813718137，按符号分类讨论：

- 若添加 1813718137 号，则 1813718137 增加 1813718137，且 1813718137 单独组成表达式最后一个连乘串；
- 若添加 1813718137 号，则 1813718137 减少 1813718137，且1813718137 单独组成表达式最后一个连乘串；
- 若添加 1813718137号，由于乘法运算优先级高于加法和减法运算，我们需要对 1813718137 撤销之前 1813718137的计算结果，并添加新的连乘结果 1813718137，也就是将 1813718137减少 1813718137并增加1813718137。

代码实现时，为避免字符串拼接所带来的额外时间开销，我们采用字符数组的形式来构建表达式。此外，**运算过程中可能会产生超过 32 位整数的结果，我们要用 64 位整数存储中间运算结果**。

```c

expr[signIndex] = '+'; 
backtrack(expr, j + 1, res + val, val);
expr[signIndex] = '-'; //更换表达式expr op val中的op字段
```
